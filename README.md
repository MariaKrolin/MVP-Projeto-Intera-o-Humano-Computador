# MVP-Projeto-Intera-o-Humano-Computador
# 🏋️ SeuRitmo - Aplicativo de Acompanhamento de Treinos

Projeto desenvolvido para a disciplina de **Interação Humano-Computador (IHC)**, com o objetivo de aplicar os conceitos de Modelagem da Interação (MoLIC) e Prototipação de Interfaces.

---

## 📖 Sobre o projeto

O **SeuRitmo** é um aplicativo mobile desenvolvido para auxiliar alunos que realizam treinos acompanhados por um personal trainer.

O aplicativo permite que o aluno acompanhe seu treino diário, registre sua execução e solicite alterações no plano de treinamento quando necessário, proporcionando uma comunicação mais eficiente entre aluno e personal trainer.

O projeto contempla desde a definição das personas até a modelagem da interação e o desenvolvimento do protótipo de alta fidelidade.

---

## 🎯 Objetivos do projeto

Foram modelados dois objetivos principais do usuário:

### 1. Realizar o treino do dia

O usuário:

- visualiza o treino prescrito pelo personal;
- consulta vídeos de execução quando possui dúvidas;
- registra cargas, repetições;
- recebe confirmação do registro do treino.

---

### 2. Solicitar alteração do treino

O usuário pode solicitar alterações ao personal trainer em dois contextos:

- informar uma limitação física;
- informar uma mudança de objetivo.

Após o envio, o sistema processa a solicitação e confirma seu recebimento.

---

## 👥 Personas

O projeto foi desenvolvido considerando duas personas pertencentes ao mesmo papel de usuário (aluno da academia).

### Júlio César

Aluno iniciante que utiliza o aplicativo para:

- consultar seu treino;
- aprender a executar corretamente os exercícios;
- registrar o treino realizado.

---

### Fernanda Lima

Aluna intermediária que utiliza o aplicativo para:

- realizar seus treinos de forma independente;
- solicitar alterações no treino;
- comunicar limitações físicas ou mudanças de objetivo ao personal.

---

## 🗺 Modelo de Interação (MoLIC)

O diagrama MoLIC representa todas as conversas entre usuário e sistema para atingir os dois objetivos definidos.

O modelo contempla:

- tópicos de conversa para cada objetivo;
- processamento do sistema;
- conversas alternativas;
- desvios temporários de tópico;
- recuperação de rupturas;
- consistência entre fluxos semelhantes.

---

## 🎨 Protótipo

Foi desenvolvido um protótipo de **alta fidelidade (UI)** para dispositivos móveis utilizando o **Figma**.

O fluxo prototipado contempla o objetivo:

> **Realizar o treino do dia**

Incluindo:

- tela inicial;
- acesso ao treino;
- consulta à execução do exercício;
- registro do treino;
- confirmação;
- telas de erro previstas no MoLIC.

---

## 🧩 Componentes utilizados

O protótipo utiliza componentes inspirados nas Human Interface Guidelines da Apple.

Entre eles:

- Status Bar
- Bottom Navigation
- Cards
- Botões Primários
- Chips
- Avatar
- Ícones
- Listas
- Badges
- Indicadores de progresso
- Alertas
- Feedback visual

Também foram aplicadas microinterações como:

- feedback ao toque;
- mudança de estado dos botões;
- animações de navegação;
- indicadores de progresso.
